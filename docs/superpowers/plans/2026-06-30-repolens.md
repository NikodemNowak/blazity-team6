**Goal:** A Next.js web app that ingests a (public or private) GitHub repo and lets a developer chat with it, generate onboarding docs from it, and drift-check its docs against the code â€” every AI answer backed by `file:line` citations rendered from the real source.

**Architecture:** Single Next.js (App Router) app. A server-side **core** fetches a repo via the GitHub REST API into an in-memory bundle keyed by `repoId`; three feature route handlers (`/api/chat`, `/api/generate-docs`, `/api/drift`) each consume that bundle and call Claude through one shared wrapper. The UI is one page with a repo loader + three tabs. No database. Deployed via Cloudflare Tunnel to `nikodem.monster`.

**Tech Stack:** Next.js 15 (App Router), TypeScript, React, Tailwind CSS, `@anthropic-ai/sdk`, GitHub REST API (`fetch`), Cloudflare Tunnel (`cloudflared`).

## Global Constraints

- **Stack:** Next.js + TypeScript, App Router. Package manager **npm**.
- **AI provider:** Claude via `@anthropic-ai/sdk`, `new Anthropic()` reading `ANTHROPIC_API_KEY` from env. Models: **`claude-sonnet-4-6`** for chat, **`claude-opus-4-8`** for docs & drift. Adaptive thinking: `thinking: { type: "adaptive" }`. **Never** pass `budget_tokens`, `temperature`, `top_p`, `top_k` (400 on these models).
- **Secrets server-side only:** `ANTHROPIC_API_KEY` and `GITHUB_TOKEN` live in `.env.local`, never `NEXT_PUBLIC_*`, never returned to the client, never logged. `.env.local` is gitignored.
- **GitHub auth:** fine-grained PAT in `GITHUB_TOKEN`, `Contents: Read-only`, scoped to the demo repo(s). Headers on every call: `Authorization: Bearer ${GITHUB_TOKEN}`, `Accept: application/vnd.github+json`, `X-GitHub-Api-Version: 2022-11-28`.
- **Ingestion budget:** skip `node_modules`, `.git`, lockfiles, and binaries; only blobs `< 100_000` bytes; cap at **300 files** and **~600_000 chars** total; record what was dropped.
- **All Claude calls are server-side** (route handlers). Citations are `{ path, startLine, endLine }`; the UI renders the snippet from the loaded bundle (the "checked, not trusted" evidence).
- **Time box:** ~3h, 3 devs. Favor a working end-to-end demo over completeness. TDD is applied to **pure functions** (filtering, parsing, citation slicing); routes and UI are verified by `curl` / browser per the steps below.

```ts
import type { RepoFile } from "./types";

const SKIP_DIRS = ["node_modules/", ".git/", "dist/", "build/", ".next/", "vendor/"];
const SKIP_FILES = [
  "package-lock.json", "yarn.lock", "pnpm-lock.yaml", "bun.lockb", "poetry.lock",
  "go.sum", "cargo.lock",
];
const BINARY_EXT = [
  ".png", ".jpg", ".jpeg", ".gif", ".webp", ".ico", ".svg", ".pdf", ".zip",
  ".gz", ".woff", ".woff2", ".ttf", ".eot", ".mp4", ".mp3", ".lock", ".min.js",
];

export function shouldIncludePath(path: string): boolean {
  const lower = path.toLowerCase();
  if (SKIP_DIRS.some((d) => lower.includes(d))) return false;
  if (SKIP_FILES.includes(lower.split("/").pop() ?? "")) return false;
  if (BINARY_EXT.some((e) => lower.endsWith(e))) return false;
  return true;
}

export function applyBudget(
  files: RepoFile[],
  maxFiles = 300,
  maxChars = 600_000,
): { kept: RepoFile[]; truncated: boolean } {
  const kept: RepoFile[] = [];
  let chars = 0;
  for (const f of files) {
    if (kept.length >= maxFiles || chars + f.content.length > maxChars) {
      return { kept, truncated: true };
    }
    kept.push(f);
    chars += f.content.length;
  }
  return { kept, truncated: false };
}
```

```ts
import { shouldIncludePath } from "./filter";
import type { RepoFile } from "./types";

const H = {
  Authorization: `Bearer ${process.env.GITHUB_TOKEN ?? ""}`,
  Accept: "application/vnd.github+json",
  "X-GitHub-Api-Version": "2022-11-28",
};

export function parseRepoUrl(url: string): { owner: string; repo: string } {
  const m = url.trim().replace(/\.git$/, "").match(/github\.com[/:]([^/]+)\/([^/]+)/);
  if (!m) throw new Error("Not a GitHub repo URL");
  return { owner: m[1], repo: m[2] };
}

async function getDefaultBranch(owner: string, repo: string): Promise<string> {
  const res = await fetch(`https://api.github.com/repos/${owner}/${repo}`, { headers: H });
  if (!res.ok) throw new Error(`GitHub repo lookup failed (${res.status})`);
  const json = await res.json();
  return json.default_branch as string;
}

export async function fetchRepoFiles(
  owner: string,
  repo: string,
): Promise<{ branch: string; files: RepoFile[]; fileTree: string[] }> {
  const branch = await getDefaultBranch(owner, repo);
  const base = `https://api.github.com/repos/${owner}/${repo}`;
  const treeRes = await fetch(`${base}/git/trees/${branch}?recursive=1`, { headers: H });
  if (!treeRes.ok) throw new Error(`GitHub tree fetch failed (${treeRes.status})`);
  const tree = await treeRes.json();

  const fileTree: string[] = tree.tree
    .filter((n: any) => n.type === "blob")
    .map((n: any) => n.path);

  const candidates = tree.tree.filter(
    (n: any) => n.type === "blob" && n.size < 100_000 && shouldIncludePath(n.path),
  );

  // Throttle blob fetches to ~8 concurrent to avoid GitHub's secondary abuse limit.
  const files: RepoFile[] = [];
  const queue = candidates.slice(0, 300);
  const worker = async () => {
    while (queue.length) {
      const node = queue.shift();
      if (!node) break;
      const b = await fetch(`${base}/git/blobs/${node.sha}`, { headers: H }).then((r) => r.json());
      const content = Buffer.from(b.content ?? "", "base64").toString("utf8");
      files.push({ path: node.path, content });
    }
  };
  await Promise.all(Array.from({ length: 8 }, worker));
  files.sort((a, b) => a.path.localeCompare(b.path));

  return { branch, files, fileTree };
}
```

```ts
import Anthropic from "@anthropic-ai/sdk";
import { PROVIDER, MODELS } from "./config";
import type { RepoBundle } from "./types";

const anthropic = new Anthropic(); // reads ANTHROPIC_API_KEY

export async function askLLM(opts: {
  system: string;
  user: string;
  model?: string;
  maxTokens?: number;
  outputFormat?: object; // JSON-schema format (provider-translated)
}): Promise<string> {
  const model = opts.model ?? MODELS.chat;
  const maxTokens = opts.maxTokens ?? 8000;

  if (PROVIDER === "anthropic") {
    const res = await anthropic.messages.create({
      model,
      max_tokens: maxTokens,
      thinking: { type: "adaptive" },
      system: opts.system,
      messages: [{ role: "user", content: opts.user }],
      ...(opts.outputFormat ? { output_config: { format: opts.outputFormat } } : {}),
    } as Anthropic.MessageCreateParamsNonStreaming);
    const text = res.content.find((b) => b.type === "text");
    return text && "text" in text ? text.text : "";
  }

  // Other providers are added here â€” see the "Provider & model configuration"
  // section of the plan. Until implemented, fail loudly rather than silently.
  throw new Error(`AI_PROVIDER="${PROVIDER}" not implemented yet`);
}

export function buildRepoContext(bundle: RepoBundle): string {
  const tree = bundle.fileTree.join("\n");
  const body = bundle.files
    .map((f) => {
      const numbered = f.content
        .split("\n")
        .map((line, i) => `${i + 1}\t${line}`)
        .join("\n");
      return `=== FILE: ${f.path} ===\n${numbered}`;
    })
    .join("\n\n");
  return `Repository: ${bundle.owner}/${bundle.repo} (branch ${bundle.branch})\n\nFILE TREE:\n${tree}\n\nFILE CONTENTS (each line prefixed with its 1-indexed line number):\n${body}`;
}
```

```ts
import type { RepoBundle, Citation } from "./types";

export function getSnippet(
  bundle: RepoBundle,
  c: Citation,
): { lines: { n: number; text: string }[] } | null {
  const file = bundle.files.find((f) => f.path === c.path);
  if (!file) return null;
  const all = file.content.split("\n");
  const start = Math.max(1, c.startLine);
  const end = Math.min(all.length, c.endLine);
```

```ts
import { NextRequest, NextResponse } from "next/server";
import { parseRepoUrl, fetchRepoFiles } from "@/lib/github";
import { applyBudget } from "@/lib/filter";
import { saveBundle } from "@/lib/bundleStore";
import type { IngestResponse, RepoBundle } from "@/lib/types";

export const runtime = "nodejs"; // needs Buffer + module-level Map

export async function POST(req: NextRequest) {
  try {
    const { repoUrl } = await req.json();
    if (!repoUrl) return NextResponse.json({ error: "repoUrl is required" }, { status: 400 });

    const { owner, repo } = parseRepoUrl(repoUrl);
    const { branch, files, fileTree } = await fetchRepoFiles(owner, repo);
    const { kept, truncated } = applyBudget(files);

    const bundle: RepoBundle = {
      repoId: `${owner}/${repo}`,
      owner, repo, branch,
      files: kept,
      fileTree,
      stats: {
        filesLoaded: kept.length,
        filesTotal: fileTree.length,
        totalChars: kept.reduce((n, f) => n + f.content.length, 0),
        truncated,
      },
    };
    saveBundle(bundle);

    const body: IngestResponse = { repoId: bundle.repoId, fileTree, stats: bundle.stats };
    return NextResponse.json(body);
  } catch (e: any) {
    return NextResponse.json({ error: e?.message ?? "Ingest failed" }, { status: 500 });
  }
}
```

```ts
  type: "json_schema",
  schema: {
    type: "object",
    properties: {
      answer: { type: "string" },
      citations: {
        type: "array",
        items: {
          type: "object",
          properties: {
            path: { type: "string" },

          additionalProperties: false,

    additionalProperties: false,

  if (!bundle) return NextResponse.json({ error: "Repo not loaded" }, { status: 404 });

    "Every citation's path must appear in the file tree and lines must exist. " +
```

```ts
  });

  let parsed: DriftResponse;
  try {
    parsed = JSON.parse(raw);
  } catch {
    parsed = { findings: [] };
  }
  return NextResponse.json(parsed);
}
```

## Provider & model configuration (extensibility)

The default and demo provider is **Claude** (we hold the event API key). The app is built so the AI backend and per-task model are **configurable by env**, and so another popular provider can be added behind the same `askLLM` interface without touching any feature code.

### What is configurable (no code change)

All read in `src/lib/config.ts`; set these in `.env.local`:

| Env var | Default | Purpose |
|---|---|---|
| `AI_PROVIDER` | `anthropic` | Which backend `askLLM` routes to (`anthropic` \| `openai` \| `google`) |
| `MODEL_CHAT` | `claude-sonnet-4-6` | Model for the Chat tab (fast, interactive) |
| `MODEL_DOCS` | `claude-opus-4-8` | Model for doc generation (hardest reasoning) |
| `MODEL_DRIFT` | `claude-opus-4-8` | Model for drift detection (hardest reasoning) |
| `MAX_TOKENS_CHAT` / `_DOCS` / `_DRIFT` | 4000 / 8000 / 6000 | Output caps per task |

### Why these model choices

| Task | Default | Rationale |
|---|---|---|
| **Chat** | `claude-sonnet-4-6` | Interactive â€” latency matters; Sonnet is fast and strong enough for grounded Q&A over a repo bundle. |
| **Docs** | `claude-opus-4-8` | One-shot, quality-sensitive long-form output over the whole repo â€” worth the strongest model. |
| **Drift** | `claude-opus-4-8` | Hardest reasoning (find a doc claim that the code contradicts, with an exact citation) â€” accuracy over speed. |

Want it cheaper/faster across the board: set `MODEL_DOCS`/`MODEL_DRIFT` to `claude-sonnet-4-6`, or `MODEL_CHAT` to `claude-haiku-4-5`. Want max quality: set all three to `claude-opus-4-8`.

### Adding another popular provider (env-switchable)

Each provider is one branch in `src/lib/llm.ts`. The contract is fixed: take `{ system, user, model, maxTokens, outputFormat? }`, return the response **text** (the JSON string when `outputFormat` is set). Features never change.

**OpenAI** (`AI_PROVIDER=openai`):
```bash
npm install openai
# .env.local:  OPENAI_API_KEY=...  AI_PROVIDER=openai
#              MODEL_CHAT=gpt-5.1-mini  MODEL_DOCS=gpt-5.1  MODEL_DRIFT=gpt-5.1
```
Add to `llm.ts`:
```ts
import OpenAI from "openai";
const openai = new OpenAI();
// inside askLLM, replacing the throw:
if (PROVIDER === "openai") {
  const res = await openai.chat.completions.create({
    model, max_completion_tokens: maxTokens,
    messages: [
      { role: "system", content: opts.system },
      { role: "user", content: opts.user },
    ],
    // structured output: OpenAI uses response_format, not output_config
    ...(opts.outputFormat
      ? { response_format: { type: "json_schema", json_schema: { name: "out", schema: (opts.outputFormat as any).schema, strict: true } } }
      : {}),
  });
  return res.choices[0]?.message?.content ?? "";
}
```

**Google Gemini** (`AI_PROVIDER=google`):
```bash
npm install @google/genai
# .env.local:  GOOGLE_API_KEY=...  AI_PROVIDER=google
#              MODEL_CHAT=gemini-2.5-flash  MODEL_DOCS=gemini-2.5-pro  MODEL_DRIFT=gemini-2.5-pro
```
Add a `PROVIDER === "google"` branch using `@google/genai`'s `generateContent`, passing `systemInstruction`, the user text, and (for structured tasks) `responseMimeType: "application/json"` + `responseSchema`. Return `response.text`.

> **Caveats when switching:** (1) Structured-output shape differs per provider (`output_config.format` on Claude, `response_format` on OpenAI, `responseSchema` on Gemini) â€” that's exactly why it lives behind `askLLM`; the `FORMAT` JSON schema in the chat/drift routes is reused as the source schema. (2) `thinking: {type:"adaptive"}` is Claude-only â€” other branches must not send it. (3) Confirm the exact current model IDs for the chosen provider before relying on them; the strings above are examples. (4) Keep every provider key server-side only (never `NEXT_PUBLIC_*`).

Verification cheat for any provider: set the env vars, restart `npm run dev`, run one Chat question â€” if it answers with citations, the branch works.
